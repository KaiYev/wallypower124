# Мой проект (masonry infinite-scroll)

Мини-гайд, чтобы с нуля разобраться с Git/GitHub и выложить этот сайт на свой `.studio` домен.

## 1. Что такое Git и GitHub (коротко)

- **Git** — программа на твоём компьютере, которая запоминает историю изменений файлов (кто, что и когда поменял). Работает локально, без интернета.
- **GitHub** — сайт, где хранится "облачная копия" твоего git-проекта (репозитория). Плюс бесплатный хостинг статичных сайтов — GitHub Pages.
- **Репозиторий (repo)** — папка проекта, за которой следит git.
- **Commit** — сохранённый снимок изменений с комментарием ("что я сделал").
- **Push** — отправить свои commit'ы с компьютера на GitHub.
- **Pull** — забрать изменения с GitHub к себе.

## 2. Установка (один раз)

1. Поставь Git: https://git-scm.com/downloads
2. Зарегистрируйся на https://github.com
3. Настрой git в терминале:
   ```
   git config --global user.name "Твоё Имя"
   git config --global user.email "твой@email.com"
   ```

## 3. Создать репозиторий на GitHub

1. На github.com жми **New repository**.
2. Назови его, например, `my-studio-site`.
3. НЕ добавляй README при создании (он уже есть у нас) — просто создай пустой репозиторий.
4. Скопируй ссылку репозитория (вида `https://github.com/username/my-studio-site.git`).

## 4. Залить эту папку на GitHub

Открой терминал в этой папке (`site-project`) и выполни по очереди:

```
git init
git add .
git commit -m "первая версия сайта"
git branch -M main
git remote add origin https://github.com/USERNAME/my-studio-site.git
git push -u origin main
```

После этого файлы появятся в репозитории на GitHub.

## 5. Включить GitHub Pages (бесплатный хостинг)

1. В репозитории → **Settings → Pages**.
2. Source: выбери ветку `main`, папку `/ (root)`.
3. Сохрани — через минуту сайт будет доступен по адресу
   `https://username.github.io/my-studio-site/`.

## 6. Подключить свой домен .studio

1. Купи домен на Namecheap / Cloudflare Registrar / Reg.ru (набери `твоеназвание.studio`).
2. В настройках DNS домена добавь записи, которые укажет GitHub (Settings → Pages → Custom domain):
   - обычно это A-записи на IP GitHub Pages, либо CNAME-запись на `username.github.io`.
3. В файле `CNAME` (уже лежит в этой папке) замени содержимое на свой реальный домен, например:
   ```
   moifashion.studio
   ```
4. Закоммить и запушь изменение:
   ```
   git add CNAME
   git commit -m "подключаю домен"
   git push
   ```
5. Подожди 10–60 минут, пока DNS обновится — и сайт будет открываться по твоему домену.

## 7. Как обновлять сайт дальше

Каждый раз, когда меняешь `index.html` или добавляешь картинки:

```
git add .
git commit -m "что изменил"
git push
```

Через минуту-две изменения появятся на живом сайте.

## 8. Своя админка (admin.html)

В проекте есть `admin.html` — твоя личная панель публикации. Она пишет
прямо в GitHub через API, без сервера.

**Настройка (один раз):**

1. Открой `admin.html` в браузере.
2. Заполни:
   - **GitHub username** — твой логин на GitHub.
   - **Название репозитория** — как назвал репозиторий (например `site-project`).
   - **Personal Access Token** — создать: GitHub → фото профиля (правый
     верхний угол) → **Settings** → в самом низу слева **Developer settings**
     → **Personal access tokens** → **Tokens (classic)** → **Generate new token**
     → поставь галочку **repo** → **Generate token** → скопируй значение
     (начинается на `ghp_...`, показывается только один раз).
3. Жми **"Сохранить подключение"** — токен остаётся только в твоём браузере.

**Публикация поста:**

1. Пиши подпись, выбираешь стиль плитки и (по желанию) штамп типа "EXCLUSIVE".
2. Можно сразу загрузить картинку — попадёт в папку `images/` репозитория.
3. Жмёшь **"Опубликовать"** — админка сама:
   - заливает картинку в репозиторий (если была),
   - обновляет `content.json` новой записью,
   - коммитит изменения на GitHub.
4. Через 30-60 секунд GitHub Pages пересоберёт сайт, и пост появится на
   `index.html` — самым первым (новые сверху).

**Важно:** токен даёт полный доступ к твоим репозиториям. Не выкладывай
`admin.html` с уже сохранённым токеном публично и никому не пересылай сам токен.
Если случайно засветил — зайди в GitHub Settings → Personal access tokens и удали его.

## 9. Как это устроено внутри

- `content.json` — просто список постов в JSON. Можно и руками отредактировать
  через GitHub (открыть файл в репозитории → карандаш "Edit" → сохранить).
- `index.html` при загрузке делает `fetch('content.json')` и рисует плитки
  на основе того, что там лежит — без рандома, это уже настоящий контент.
- `admin.html` — просто удобная форма поверх GitHub API, чтобы не редактировать
  JSON руками каждый раз.

## Дальше — куда развивать самому

- Добавить в `admin.html` поле "категория" или "тег" и фильтр на сайте.
- Добавить удаление/редактирование постов (сейчас админка только добавляет).
- Добавить `editor.html` (визуальный GrapesJS-редактор) для вёрстки — уже есть
  в проекте, можно использовать параллельно для экспериментов с дизайном.
- Кнопку "лайк"/просмотры — понадобится либо GitHub API (как сейчас), либо
  простая база данных (например Supabase, тоже бесплатно для старта).
